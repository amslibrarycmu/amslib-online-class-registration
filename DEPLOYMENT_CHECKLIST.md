# เช็คลิสต์ก่อนนำระบบขึ้นเซิร์ฟเวอร์จริง (Deployment Checklist)

เพื่อให้มั่นใจว่าระบบ AMSLIB Online Class Registration จะทำงานได้อย่างสมบูรณ์เมื่อย้ายจาก Localhost ไปยัง Server จริง (Production) กรุณาตรวจสอบ 4 หัวข้อนี้ก่อนเริ่มใช้งาน:

## 1. การอัปโหลดไฟล์ Dependencies (โฟลเดอร์ vendor)
ระบบมีการใช้ **PHPMailer** เพื่อส่งอีเมลยืนยันการลงทะเบียน ซึ่งติดตั้งผ่าน Composer
- [ ] ต้องอัปโหลดโฟลเดอร์ `vendor` ที่อยู่ภายในโฟลเดอร์ `backend-raw` ขึ้นไปยังเซิร์ฟเวอร์ด้วย 
- *(หรือหากเซิร์ฟเวอร์มี Composer สามารถรันคำสั่ง `composer install` ภายในโฟลเดอร์ `backend-raw` บนเซิร์ฟเวอร์แทนได้)*

## 2. การตั้งค่าฐานข้อมูล (Database) ในไฟล์ `.env`
ไฟล์ `.env` ในโฟลเดอร์ `backend-raw` ต้องถูกอัปเดตให้ตรงกับฐานข้อมูลของเซิร์ฟเวอร์จริง
- [ ] `DB_HOST`: เปลี่ยนจาก `mysql` (หรือ `localhost`) เป็น Host ของฐานข้อมูลบนเซิร์ฟเวอร์จริง
- [ ] `DB_USER`: ชื่อผู้ใช้งานฐานข้อมูลบนเซิร์ฟเวอร์จริง
- [ ] `DB_PASSWORD`: รหัสผ่านฐานข้อมูลบนเซิร์ฟเวอร์จริง
- [ ] `DB_DATABASE`: ชื่อฐานข้อมูล (หากมีการเปลี่ยนแปลง)

## 3. การตั้งค่า Microsoft Entra ID (ระบบล็อกอิน)
เพื่อให้ระบบล็อกอินด้วยบัญชีมหาวิทยาลัยทำงานได้ถูกต้องบนโดเมนจริง
- [ ] ตรวจสอบว่า `MS_ENTRA_ID_REDIRECT_URI` ในไฟล์ `.env` ตรงกับโดเมนจริง (ปัจจุบันตั้งค่าเป็น `https://library.ams.cmu.ac.th/library/amslibclass/backend/public/api/auth/callback` ซึ่งถูกต้องแล้ว)
- [ ] เข้าไปที่ **Azure Portal** (Microsoft Entra ID) ของมหาวิทยาลัย และเพิ่มลิงก์ Redirect URI ข้างต้นลงในเมนู Authentication ของแอปพลิเคชัน เพื่ออนุญาตให้โดเมนจริงทำการล็อกอินได้

## 4. การ Build ฝั่ง Frontend (React)
ก่อนอัปโหลดฝั่งหน้าเว็บ (โฟลเดอร์ `frontend`) ขึ้นเซิร์ฟเวอร์ ต้องทำการ Build โค้ดให้เป็นเวอร์ชันใช้งานจริง
- [ ] ในไฟล์ `.env` ของฝั่ง `frontend` ต้องกำหนดค่า `VITE_API_URL` ให้ชี้ไปยังโดเมนของ Backend ตัวจริง
- [ ] รันคำสั่ง Build (เช่น `npm run build`) ในโฟลเดอร์ `frontend`
- [ ] นำไฟล์ทั้งหมดที่ได้จากโฟลเดอร์ `dist` (หรือ `build`) ไปอัปโหลดขึ้น Web Server (เช่น ภายในโฟลเดอร์ `public_html` หรือ `htdocs`)
